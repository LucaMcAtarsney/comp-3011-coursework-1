    POST /runs/start

    Starts a new game run. This endpoint handles both new and existing players. If create_new_player is true, a new player is created. Otherwise, the existing player is authenticated.

    Body parameter

    {
  "player_name": "string",
  "password": "string",
  "map_id": "string",
  "create_new_player": false
}

    Parameters

    Name In Type Required Description
    body body RunStart true none

    Example responses

    200 Response

    {
  "player_id": 0,
  "run_id": 0
}

    Responses

    Status Meaning Description Schema
    200 OK Successful Response RunStartResponse
    422 Unprocessable Entity Validation Error HTTPValidationError
    This operation does not require authentication
